## 슬립 방지 (녹음 중)

**현황**: 녹음 시작/종료 시 슬립 방지 처리가 없음. 시스템 슬립 진입 시 CoreAudio 스트림이 끊겨 녹음이 중단되고, 깨어나도 자동 복구되지 않음. 화면 슬립(display sleep)만 들어가는 경우는 보통 무관하나 App Nap throttle 위험은 존재.

**해결안**:

1. (1순위) 녹음 시작 시 `caffeinate -i` 자식 프로세스 spawn → 종료/일시정지 시 kill. 단순, 외부 의존 없음.
2. `IOPMAssertionCreateWithName` FFI 직접 호출. 더 깔끔하지만 unsafe + objc 바인딩 필요.

**보류 사유**: 노트 테이킹 중에는 화면을 켜둔 채 쓰기 때문에 즉시 필요하지 않음. 백그라운드 장시간 녹음 케이스 발생 시 1번으로 처리.
